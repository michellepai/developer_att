1. Introduction

The In App Messaging API sends and retrieves SMS and MMS messages on behalf of a customer if that customer has provided user consent authorization. When sending a message on behalf of a customer, that customer's phone number (MSISDN) will be retrieved from the customer's consent information. The message is then forwarded to the recipients using the customer's MSISDN as the sender address. A developer is able to make a single request to the API Gateway and the AT&T servers determine if the request is an SMS message or an MMS message and select the correct transport mechanism accordingly.

  • Sending messages from your app: The Send Message method sends text and picture messages to any mobile phone in the USA using their own AT&T Wireless phone number as the sender address, from within your app. Message recipients are able to immediately recognize who sent the message.
  • Receiving messages in your app: Responses to the sent messages are able to be received by the user's mobile phone or by your app using the Get Message Headers method and Get Message Contents method. Your app may display messages received by the user's mobile phone, allowing the conversation to continue from within your app.
Considerations

The RESTful In-App Messaging API provided by AT&T has the following considerations for you to keep in mind.

For all of the In-App Messaging API methods.

  • The consent from the customer is required.
  • The methods are only available to AT&T Wireless post-paid mobile phone customers.

For both of the Get Message Contents method and the Get Message Headers method.

  • In order to receive messages in the app using the Get Message Contents method and Get Message Headers method, the message sender must be provisioned with an AT&T Messages inbox. Sign up for an AT&T Messages inbox at messages.att.net.

For the Send Message method.

  • This method may send messages to mobile devices on other carriers in the USA.
Provisioning

The RESTful In-App Messaging API provided by AT&T requires that you provision your app as follows.

You must register the In-App Messaging API for your app account on the AT&T Developer Program web site in order to utilize this API.

2. OAuth

The APIs provided by AT&T use the OAuth 2.0 framework, and an OAuth access token must be included in each request to the API Gateway. To obtain an OAuth access token, use the App Key and App Secret from the My Apps section after registering and signing into your API Platform account.

You may authorize access for this API by using the following scopes.

  • "IMMN" for the Send Message method.
  • "MIM" for the Get Message Contents method and the Get Message Headers method.

To obtain user consent authorization run the following command from the Web browser and replace "{{APP_KEY}}" and "{{API_SCOPES}}" with the App Key and the API scopes that you wish to access as follows.

Get User Authorization method using the HTTP request:

copy 
https://api.att.com/oauth/authorize?client_id={{APP_KEY}}&scope={{API_SCOPES}}&redirect_uri={{REDIRECT_URI}}

A successful response from the API Gateway opens the User Consent Authorization page.

After the user has completed the User Consent Authorization page, the API Gateway sends a response.

Extract the value from "{{OAUTH_AUTHORIZATION_CODE}}" to use in the Get Access Token request to the API Gateway.

Response URI for successful OAuth Authorization Code:

https://{{YOUR_OAUTH_URI}}?code={{OAUTH_AUTHORIZATION_CODE}}

After obtaining the OAuth authorization code run the following command from the Command Line Interface window and replace "{{APP_KEY}}", "{{APP_SECRET}}", and "{{OAUTH_AUTHORIZATION_CODE}}" with the App Key, App Secret, and the OAuth authorization code as follows.

Get Access Token method using the cURL command:

copy 
curl "https://api.att.com/oauth/token" \
    --insecure \
    --header "Accept: application/x-www-form-urlencoded" \
    --header "Content-Type: application/x-www-form-urlencoded" \
    --data "grant_type=authorization_code&client_id={{APP_KEY}}&client_secret={{APP_SECRET}}&code={{OAUTH_AUTHORIZATION_CODE}}"

A successful response from the API Gateway contains an OAuth access token, the expiration period, and a refresh token as displayed in the following example.

Response from the API Gateway for the Get Access Token method:

{
    "access_token":"{{OAUTH_ACCESS_TOKEN}}",
    "token_type": "bearer",
    "expires_in":"{{EXPIRATION_PERIOD}}",
    "refresh_token":"{{REFRESH_TOKEN}}"
}

Extract the value from "{{OAUTH_ACCESS_TOKEN}}" to use in the Authorization parameter for requests to the API Gateway. When sending an request to the API Gateway a successfully constructed Authorization parameter looks like the following.

Authorization parameter for the HTTP header:

Authorization: Bearer {{OAUTH_ACCESS_TOKEN}}

For more information about refresh_token and other authentication options, OAuth 2.0 API.

3. Resources
Get Message Contents
Overview

The Get Message Contents method retrieves media for one or more customer messages from the AT&T Messages app environment using information returned by the Get Message Headers method request.

  • OAuth Scope

  • Scope: MIM

  • Model: auth_code

  • Resource
  • copy 
    /MyMessages/{{MessageId}}/{{PartNumber}}
Examples

Request

copy 
GET /rest/1/MyMessages/WU124/1
Host: api.att.com
Authorization: Bearer 31dbab574c01ac901af7d6140574ea8

Response

copy 
HTTP/1.1 200 OK 
Content-Type: image/jpeg 
Content-Length: nnnn 
Date: Thu, 04 Jun 2009 02:51:59 GMT

...JPEG binary image data...
Request Parameters
Parameter
Accept String Header
Specifies the format of the body for the response. The acceptable values for this parameter are:
  • application/json
  • application/xml
The default value is application/json.
Authorization String Header
Specifies the authorization. The acceptable format for this parameter is the word "Bearer" followed by a space ( ) and an OAuth access token. If this parameter value is missing from the header, then the API Gateway returns a message with an HTTP status code of 400 Invalid Request. If the OAuth access token is not valid, then the API Gateway returns an HTTP status code of 401 Unauthorized with a WWW-Authenticate HTTP header.
MessageId String ResourceURL
Specifies the message identifier (ID) that represents a customer message in the AT&T Messages app environment.
PartNumber String ResourceURL
Specifies the content ID that represents an attachment in the referenced customer message.
Response Parameters
Parameter
Content-Length Integer Header
Specifies the the length, in octets, of the HTTP body of the response. An octet is an 8-bit byte.
Content-Transfer-Encoding String Header
Specifies the encoding in which the entity body was transferred. This parameter should be used when the content is encoded in base64.
Content-Type String Header
Specifies the representation format of the HTTP body of the response.

Request Parameters
Parameter
Accept String Header
Specifies the format of the body for the response. The acceptable values for this parameter are:
  • application/json
  • application/xml
The default value is application/json.
Authorization String Header
Specifies the authorization. The acceptable format for this parameter is the word "Bearer" followed by a space ( ) and an OAuth access token. HeaderCount Integer QueryString
Specifies the number of headers being returned. The acceptable values for this parameter are:
  • 1 through 500

Specifies the index value for which the HeaderCount parameter begins the count. This parameter value is an opaque string used as a cursor for pagination. Specifies the part number content ID for the message.
Send Message
Overview

The Send Message method sends messages on behalf of an AT&T Wireless customer. Each time the Send Message method request is sent, a single message may be sent to one or more destination mobile devices. Messages are processed synchronously and sent asynchronously to the destination mobile device.

  • This method sends messages as SMS or MMS as follows.
    • For SMS: Text only, using the ISO-8859-1 format. The acceptable maximum length is 160 characters.
    • For MMS messages: Any attachment, or text with a length greater than 160 characters.
  • This method supports the following MMS content types.
    • application/vnd.wap.multipart.mixed - without SMIL component.
    • application/vnd.wap.multipart.related - without SMIL component.
    • application/vnd.wap.multipart.related - with SMIL component.
  • This method is limited to the following content types for MMS messages.
    • ASCII
    • UTF-8
    • UTF-16
    • ISO-8859-1
    • JPEG
    • GIF87a
    • GIF89a
    • 3GPP
    • MP4
    • AMR-NB
    • SP-MIDI
    • MP3
    • AAC-LC
    • V-card PIM
    • PIM Support - vCalendar 1.0
  • OAuth Scope

  • Scope: IMMN

  • Model: auth_code

Request Parameters
Parameter
Accept String Header
Specifies the format of the body for the response. The acceptable values for this parameter are:
  • application/json
  • application/xml
The default value is application/json.
Authorization String Header
Specifies the authorization. The acceptable format for this parameter is the word "Bearer" followed by a space ( ) and an OAuth access token. If this parameter value is missing from the header, then the API Gateway returns a message with an HTTP status code of 400 Invalid Request. If the OAuth access token is not valid, then the API Gateway returns an HTTP status code of 401 Unauthorized with a WWW-Authenticate HTTP header.
Content-Type String Header
Specifies the representation format for the request. The acceptable values for this parameter are:
  • application/json
  • application/xml
  • application/x-www-form-urlencoded
Addresses String Body
Specifies one or more destination addresses. The acceptable format for this parameter is the protocol identifier (ID) followed by the URL-escaped AT&T mobile number, short code, or e-Mail address; such as tel%3A%2B16309700001, tel%3A16309700001, tel%3A6309700001, or short%3A309. The country code and preceding plus ( + ) symbol are optional. Note: The maximum number of addresses is 10 that is supported. If any address is duplicated, then the request is sent only once to that address.
Text String Body
Specifies the text portion of the message being sent. The acceptable formats for this parameter are:
  • ASCII
  • UTF-8
  • UTF-16
  • ISO-8859-1
Note: If the Attachments parameter is not specified, then this parameter must be specified.
Attachments MIME Data Body
Specifies the logical reference to one or more attachments. Multiple attachments are supported as long as the total size of the request is less than 580 kilobytes. Note: If the Text parameter is not specified, then this parameter must be specified.
Subject String Body
Specifies the subject for the message. The maximum length for this parameter is 512 characters. The acceptable values for this parameter are:
  • ASCII
  • UTF-8
  • UTF-16
  • ISO-8859-1
Group Boolean Body
Indicates the visibility of the recipients for message. If this parameter is set to true, then all recipients are visible and Reply-All is allowed. This parameter is only applicable if there are multiple recipients. If this parameter is not set, then the message is treated as Broadcast.
Response Parameters
Parameter
Id String Body
Specifies the unique identifier for the message that may be used for troubleshooting.

using System;
using System.Collections.Specialized;
using System.IO;
using System.Net;
using System.Net.Security;
using System.Security.Cryptography.X509Certificates;
using System.Text;
using System.Web;
using System.Web.Script.Serialization;


/*
 * PREREQUISITE:  Use .Net C# Web Site Project.
 * An ASP.NET page is typically divided into two files:
 *         A markup file with a .aspx extension
 *         A code file, with a .aspx.cs extension
 * This code defines a partial class named IMMN (Class name
 * can be replaced with the name of the page happens to be for you).
 * For more information about OAuth or IMMN Service parameters, refer to online 
 * documentation at https://developer.att.com.
 */
public partial class IMMN : System.Web.UI.Page
{
    string apiResponseString = string.Empty;   //successful response of api request
    string apiErrorResponse = string.Empty;    //error response of api request

    /* The Page_Load event is triggered when a page loads, and ASP.NET will 
     * automatically call the subroutine Page_Load, and execute the code 
     * inside it.
     */

    protected void Page_Load(object sender, EventArgs e)
    {
        /*If the application is deployed behind the proxy, add the value
        * of the proxy IP and Port, otherwise leave it as it is.
        */
        string proxyIp = "";
        int proxyPort = 0;

        //BypassCertificateError();  //Uncomment to ignore one way ssl handshake errors.

        // Get Authorization Code Start
        string oAuthEndPoint = "https://api.att.com/oauth/authorize";
        string oAuthHTTPVerb = "POST";
        string oAuthAccept = "application/json";
        string oAuthContentType = "application/x-www-form-urlencoded";
        string oAuthclient_id = ""; //Update with registered IMMN Service Application Id/Key.
        string oAuthclient_secret = ""; //Update with registered IMMN Service Secret Key/Id.
        string oAuthScope = "IMMN";
        string oAuthRedirectUrl = "https://wwww.google.com";
        if (Request["Code"] == null)
        {
            Response.Redirect(string.Empty + oAuthEndPoint + "?client_id=" + oAuthclient_id + "&scope=" + oAuthScope + "&redirect_url=" + oAuthRedirectUrl);
        }

        // Get Access Token based on Authorization Code
        string oAuthAuthorizationCode = Request["Code"].ToString();
        oAuthEndPoint = "https://api.att.com/oauth/token";
        string oAuthBody = "client_id=" + HttpUtility.UrlEncode(oAuthclient_id).ToString() +
                                                "&client_secret=" + HttpUtility.UrlEncode(oAuthclient_secret).ToString() +
                                                "&grant_type=" + HttpUtility.UrlEncode("authorization_code").ToString() +
                                                "&code=" + HttpUtility.UrlEncode(oAuthAuthorizationCode).ToString();
        // Send web request and handle web response 
        HttpWebRequest webRequest = null;
        HttpWebResponse webResponse = null;
        GetAccessTokenResponse getAccessTokenResponse = null;
        try
        {
            webRequest = (HttpWebRequest)System.Net.WebRequest.Create(oAuthEndPoint);
            if (!string.IsNullOrEmpty(proxyIp))
            {
                webRequest.Proxy = new WebProxy(proxyIp, proxyPort);
            }
            webRequest.ContentType = oAuthContentType;
            webRequest.Accept = oAuthAccept;
            webRequest.Method = oAuthHTTPVerb;
            byte[] postBytes = null;
            if (!string.IsNullOrEmpty(oAuthBody))
            {
                UTF8Encoding encoding = new UTF8Encoding();

                postBytes = encoding.GetBytes(oAuthBody);
            }
            if (postBytes != null)
            {
                webRequest.KeepAlive = true;
                webRequest.ContentLength = postBytes.Length;
                Stream postStream = webRequest.GetRequestStream();
                postStream.Write(postBytes, 0, postBytes.Length);
                postStream.Close();
            }

            webResponse = (HttpWebResponse)webRequest.GetResponse();
            using (StreamReader sr2 = new StreamReader(webResponse.GetResponseStream()))
            {
                apiResponseString = sr2.ReadToEnd();
                getAccessTokenResponse = new GetAccessTokenResponse();
                JavaScriptSerializer deserializeJsonObject = new JavaScriptSerializer();
                getAccessTokenResponse = (GetAccessTokenResponse)deserializeJsonObject.Deserialize(apiResponseString, typeof(GetAccessTokenResponse));
                sr2.Close();
            }
            webRequest = null;
            webResponse.Close();
            // Get Access Token End 

            // Send IMMN Start
            string immnAddresses = "tel:+1<number>"; // Update with MSISDN Number.
            string immnSubject = "StartupGuide";
            string immnMessage = "Message";
            string mmsParameters = "Addresses=" + Server.UrlEncode(immnAddresses) +
                                    "&Subject=" + Server.UrlEncode(immnSubject) + "&Text=" + Server.UrlEncode(immnMessage);
            string immnServiceEndPoint = "https://api.att.com/rest/1/MyMessages";
            string immnServiceHTTPVerb = "POST";
            string immnServiceAccept = "application/json";
            string immnServiceAuthorization = getAccessTokenResponse.access_token;
            NameValueCollection headers = new NameValueCollection();
            headers.Add("Authorization", "Bearer " + immnServiceAuthorization);
            MsgResponseId immnResponse = null;
            webRequest = (HttpWebRequest)System.Net.WebRequest.Create(immnServiceEndPoint);
            if (!string.IsNullOrEmpty(proxyIp))
            {
                webRequest.Proxy = new WebProxy(proxyIp, proxyPort);
            }
            webRequest.Accept = immnServiceAccept;
            webRequest.Method = immnServiceHTTPVerb;
            webRequest.Headers.Add(headers);
            string boundaryToSend = "----------------------------" + DateTime.Now.Ticks.ToString("x");
            string dataToSend = string.Empty;
            dataToSend += "--" + boundaryToSend + "\r\n";
            dataToSend += "Content-Type: application/x-www-form-urlencoded; charset=UTF-8\r\nContent-Transfer-Encoding: 8bit\r\nContent-Disposition: form-data; name=\"root-fields\"\r\nContent-ID: <startpart>\r\n\r\n" + mmsParameters + "\r\n";
            UTF8Encoding immnEncoding = new UTF8Encoding();

            webRequest.ContentType = "application/x-www-form-urlencoded";
            byte[] immnPostBytes = immnEncoding.GetBytes(mmsParameters);
            webRequest.ContentLength = immnPostBytes.Length;
            Stream immnStream = webRequest.GetRequestStream();
            immnStream.Write(immnPostBytes, 0, immnPostBytes.Length);
            immnStream.Close();

            webResponse = (HttpWebResponse)webRequest.GetResponse();
            using (StreamReader sr2 = new StreamReader(webResponse.GetResponseStream()))
            {
                apiResponseString = sr2.ReadToEnd();
                immnResponse = new MsgResponseId();
                JavaScriptSerializer deserializeJsonObject = new JavaScriptSerializer();
                immnResponse = (MsgResponseId)deserializeJsonObject.Deserialize(apiResponseString, typeof(MsgResponseId));
                sr2.Close();
            }
            // Send IMMN End
        }
        catch (WebException ex)
        {
            try
            {
                if (null != ex.Response)
                {
                    using (StreamReader sr2 = new StreamReader(ex.Response.GetResponseStream()))
                    {
                        apiErrorResponse = sr2.ReadToEnd();
                        sr2.Close();
                    }
                }
            }
            catch
            {
                apiErrorResponse = "Unable to get response";
            }
        }
        catch (Exception ex)
        {
            apiErrorResponse = ex.ToString();
        }
        finally
        {
            webRequest = null;
            if (webResponse != null)
                webResponse.Close();
            webResponse = null;
        }
    }

    #region IgnoreSSLHandshakeError
    private static void BypassCertificateError()
    {
        ServicePointManager.ServerCertificateValidationCallback +=
            delegate(Object sender1, X509Certificate certificate, X509Chain chain, SslPolicyErrors sslPolicyErrors)
            {
                return true;
            };
    }
    #endregion

    // OAuth Service Class
    class GetAccessTokenResponse
    {
        public string access_token { get; set; }
        public int expires_in { get; set; }
        public string refresh_token { get; set; }
        public string token_type { get; set; }
    }
    #endregion

    #region IMMNDataStructures
    /// <summary>
    /// Response from IMMN api
    /// </summary>
    public class MsgResponseId
    {
        /// <summary>
        /// Gets or sets Message ID
        /// </summary>
        public string Id { get; set; }
    }
    #endregion
}
