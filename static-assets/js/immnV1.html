1. Introduction

The In App Messaging API sends and retrieves SMS and MMS messages on behalf of a customer if that customer has provided consent via the OAuth 2.0 Authentication Management API. When sending a message on behalf of a customer, that customer's phone number (MSISDN) will be retrieved from the customer's consent information. The message will then be forwarded to the recipients using the customer's MSISDN as the sender address. A developer is able to make a single API request and the AT&T servers determine if the request is an SMS message or an MMS message and select the correct transport mechanism accordingly.

  • Sending messages from your app: The Send Message method sends text and picture messages to any mobile phone in the USA using their own AT&T Wireless phone number as the sender address, from within your app. Message recipients are able to immediately recognize who sent the message.
  • Receiving messages in your app: Responses to the sent messages are able to be received by the user's mobile phone or by your app using the Get Message Headers and Get Message Content methods. Your app may display messages received by the user's mobile phone, allowing the conversation to continue from within your app.

Considerations

The RESTful In App Messaging API provided by AT&T has the following considerations for you to keep in mind.

  • For All of the In App Messaging API methods.
    • The consent from the customer is required.
    • The methods are only available to AT&T Wireless post-paid mobile phone customers.
  • For both of the Get Message methods.
    • In order to receive messages in the app using the Get Message Content and Get Message Header methods, the message sender must be provisioned with an AT&T Messages inbox. Sign up for an AT&T Messages inbox at messages.att.net.
  • For the Send Message method.
    • This method may send messages to mobile devices on other carriers in the USA.
Provisioning

The RESTful In App Messaging API provided by AT&T requires that you provision your app as follows.

  • You must register the In App Messaging API for your app account on the AT&T Developer Program web site in order to utilize this API.

Get Message Contents
Overview

The Get Message Contents method retrieves media for one or more customer messages from the AT&T Messages app environment using information returned by the Get Message Headers method request.

  • OAuth Scope

  • Scope: MIM

  • Model: auth_code

  • Resource
  • copy 
    /MyMessages/{{MessageId}}/{{PartNumber}}
Examples

Request

copy 
GET /rest/1/MyMessages/WU124/1
Host: api.att.com
Authorization: Bearer 31dbab574c01ac901af7d6140574ea8

Response

copy 
HTTP/1.1 200 OK 
Content-Type: image/jpeg 
Content-Length: nnnn 
Date: Thu, 04 Jun 2009 02:51:59 GMT

...JPEG binary image data...
Request Parameters
Parameter
Accept String Header
Specifies the format of the body for the response. The acceptable values for this parameter are:
  • application/json
  • application/xml
The default value is application/json.
Authorization String Header
Specifies the authorization. The acceptable format for this parameter is the word "Bearer" followed by a space ( ) and an OAuth access token. If this parameter value is missing from the header, then the API Gateway returns a message with an HTTP status code of 400 Invalid Request. If the OAuth access token is not valid, then the API Gateway returns an HTTP status code of 401 Unauthorized with a WWW-Authenticate HTTP header.
MessageId String ResourceURL
Specifies the message identifier (ID) that represents a customer message in the AT&T Messages app environment.
PartNumber String ResourceURL
Specifies the content ID that represents an attachment in the referenced customer message.
Response Parameters
Parameter
Content-Length Integer Header
Specifies the the length, in octets, of the HTTP body of the response. An octet is an 8-bit byte.
Content-Transfer-Encoding String Header
Specifies the encoding in which the entity body was transferred. This parameter should be used when the content is encoded in base64.
Content-Type String Header
Specifies the representation format of the HTTP body of the response.

Request Parameters
Parameter
Accept String Header
Specifies the format of the body for the response. The acceptable values for this parameter are:
  • application/json
  • application/xml
The default value is application/json.
Authorization String Header
Specifies the authorization. The acceptable format for this parameter is the word "Bearer" followed by a space ( ) and an OAuth access token. If this parameter value is missing from the header, then the API Gateway returns a message with an HTTP status code of 400 Invalid Request. If the OAuth access token is not valid, then the API Gateway returns an HTTP status code of 401 Unauthorized with a WWW-Authenticate HTTP header.
HeaderCount Integer QueryString
Specifies the number of headers being returned. The acceptable values for this parameter are:
  • 1 through 500
This parameter value is relative to the most recent customer message. If the IndexCursor parameter is specified, then this parameter starts with the first message defined by the IndexCursor parameter.
IndexCursor String QueryString
Specifies the index value for which the HeaderCount parameter begins the count. This parameter value is an opaque string used as a cursor for pagination. For pagination, the initial Get Message Headers method request does not require this parameter, however subsequent Get Message Headers method requests must include this parameter in order to get the next block of messages.
Response Parameters
Parameter
MessageHeadersList Objects container Body
Specifies the object that contains the Headers, IndexCursor, and HeaderCount parameter values.
Headers List of Objects Body
Specifies the object that contains the MessageId, From, To, Subject, Text, MmsContent, Received, Favorite, Read, Type, and Direction parameter values.
IndexCursor String Body
Specifies the start of the next block of messages for the current request. Note: If this parameter is set to zero (0), then the message is the end of the block.
HeaderCount String Body
Specifies the number of headers returned for this request.
MessageId String Body
Specifies the unique identifier (ID) for the message.
From String Body
Specifies the sender for the message.
To List of Objects Body
Specifies the recipients for the message. If the message is a group message, then this parameter contains multiple addresses.
Subject String Body
Specifies the subject for the message.
Text String Body
Specifies the text portion of the message. Note: If the Type parameter is set to MMS, then this parameter is empty.
MmsContent list of Objects Body
Specifies the text portion of the message. Note: If the Type parameter is set to SMSTEXT, then this parameter is empty.
Received String Body
Specifies the date and time that the message is received.
Favorite Boolean Body
Indicates that the message is a favorite.
Read Boolean Body
Indicates that the message has been read.
Type String Body
Specifies the type for the message. The acceptable values for this parameter are:
  • SMSTEXT
  • MMS
Direction String Body
Specifies the direction for the message. The acceptable values for this parameter are:
  • IN
  • OUT
ContentName String Body
Specifies the content name for the message.
ContentType String Body
Specifies the content type ID for the message.
PartNumber String Body
Specifies the part number content ID for the message.
Send Message
Overview

The Send Message method sends messages on behalf of an AT&T Wireless customer. Each time the Send Message method request is sent, a single message may be sent to one or more destination mobile devices. Messages are processed synchronously and sent asynchronously to the destination mobile device.

  • This method sends messages as SMS or MMS as follows.
    • For SMS: Text only, using the ISO-8859-1 format. The acceptable maximum length is 160 characters.
    • For MMS messages: Any attachment, or text with a length greater than 160 characters.
  • This method supports the following MMS content types.
    • application/vnd.wap.multipart.mixed - without SMIL component.
    • application/vnd.wap.multipart.related - without SMIL component.
    • application/vnd.wap.multipart.related - with SMIL component.
  • This method is limited to the following content types for MMS messages.
    • ASCII
    • UTF-8
    • UTF-16
    • ISO-8859-1
    • JPEG
    • GIF87a
    • GIF89a
    • 3GPP
    • MP4
    • AMR-NB
    • SP-MIDI
    • MP3
    • AAC-LC
    • V-card PIM
    • PIM Support - vCalendar 1.0

  • OAuth Scope

  • Scope: IMMN

  • Model: auth_code

  • Resource
  • copy
